# Drizzle migrations

SQL files in this folder are generated by **`npm run db:generate`** and applied with **`npm run db:migrate`** (see [`drizzle.config.ts`](../drizzle.config.ts) and [`database-architecture.mdc`](../.cursor/rules/database-architecture.mdc)).

## Manual rollback (forward migrations only)

`drizzle-kit` does not apply down-migrations. For emergencies, some migrations ship a **companion rollback script** named like `NNNN_slug.rollback.sql`.

Example: [`0010_right_blazing_skull.rollback.sql`](./0010_right_blazing_skull.rollback.sql) reverses [`0010_right_blazing_skull.sql`](./0010_right_blazing_skull.sql). Only run against a DB backup after verifying no feature code depends on the new tables/columns/enums.

After a manual rollback you must align **`drizzle` journal state** with reality (typically restore from backup or coordinate with ops — do not delete rows from production casually).
