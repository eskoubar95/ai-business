# Task log — Stage 4, Task 4.1 (Backend)

## Scope

Webhook HMAC + idempotent engine, Notion client/sync/parser (2025 API `dataSources.query`), approvals Server Actions, orchestration event logging and `getAgentStatus`, Vitest + READMEs, `WEBHOOK_SECRET` in `.env.example` (already present).

## Implementation notes

- **Orchestration:** All logging targets `orchestration_events` / Drizzle `orchestrationEvents` (not legacy `orchestration` table).
- **Webhook engine:** Pluggable `WebhookDeliveryAdapter` for tests; production path uses `webhook_deliveries` with statuses `pending` | `delivered` | `failed`. Duplicate `idempotency_key` skips when `delivered` or `pending`.
- **Notion:** `getNotionClient` / `resolveNotionForBusiness` decrypt first business-linked Notion MCP row (join `agents`). Sync uses credential `tasksDatabaseId` as **`data_source_id`** for `client.dataSources.query` (current `@notionhq/client`). Optional `tasksDatabaseId` field added to `lib/mcp/config.ts` for the installer UI.
- **Approvals:** `createApproval`, `approveArtifact`, `rejectArtifact` log `approval.*` events and `agent.lifecycle` when `agentId` + `businessId` are present.
- **Quality gates:** `npm run lint`, `npm test -- --run`, `npm run build` — green.
