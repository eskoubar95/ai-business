# Task 1.2 - UI foundation, shadcn preset, Neon Auth UI, Playwright

## Summary

Implemented Tailwind v4 with shadcn manual theme wiring, Neon Auth (server + API route + middleware + UI routes), app shell with auth-aware navigation, protected dashboard placeholder, Playwright smoke tests, and documentation under `app/README.md`, committed on branch `ui-foundation-auth` in worktree `.apm/worktrees/ui-foundation-auth`.

## Details

- Dependencies: `@neondatabase/auth`, Tailwind v4 (`tailwindcss`, `@tailwindcss/postcss`, `postcss`), shadcn stack (`shadcn`, `clsx`, `tailwind-merge`, `class-variance-authority`, `tw-animate-css`, `lucide-react`), `@playwright/test`.
- Added `.npmrc` with `legacy-peer-deps=true` because `@neondatabase/auth` beta pulls `peerOptional next@>=16` while the app stays on Next 15; plain `npm install` fails without it.
- `components.json` came from the requested preset path; full `shadcn init` dependency step failed on peer resolution, so styling follows shadcn manual Tailwind v4 recipe (`shadcn/tailwind.css` + theme tokens) merged with `@import "@neondatabase/auth/ui/tailwind"` per Neon docs (no duplicate `ui/css`).
- Auth: `lib/auth/server.ts`, `lib/auth/client.ts`, `app/api/auth/[...path]/route.ts`, `middleware.ts` matcher for `/account/:path*` and `/dashboard/:path*` with `loginUrl` `/auth/sign-in`.
- UI: root layout wraps `NeonAuthUIProvider`; `NavShell` (Server Component) composes client `NavShellAuth` for `SignedIn` / `SignedOut` / `UserButton`; auth/account routes use `AuthView` / `AccountView` with `generateStaticParams` from `authViewPaths` / `accountViewPaths`.
- Playwright: `playwright.config.ts` with `webServer.env` fallbacks for Neon Auth vars so smoke runs without a committed `.env.local`; `tests/smoke.spec.ts` checks `/` 200 and email input on `/auth/sign-in`.
- `next.config.ts`: `outputFileTracingRoot` set to the worktree to silence incorrect workspace-root inference when multiple lockfiles exist.
